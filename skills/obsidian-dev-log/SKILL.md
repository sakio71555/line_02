# Obsidian Dev Log Skill

## Purpose

Use this skill when updating development logs and Codex task records.

## Where To Record

- `docs/14_dev_logs/YYYY-MM-DD.md`: what changed, commands, result, risks, next Loop.
- `docs/11_codex_tasks/*.md`: Codex-facing task record and verification notes.

## Rules

- Obsidian integration features are not part of the product.
- Do not create `.obsidian/`.
- Do not add Obsidian plugins, Canvas files, or app integration code.
- Do not write real customer information.
- Do not write LINE userId.
- Do not write API keys, env values, Supabase project refs, or production logs.

## Expected Result

Keep the log short enough to scan, but specific enough to see what changed and what remains risky.
